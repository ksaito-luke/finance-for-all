# 漫画 生成プロンプト 標準ガイド（全エピソード共通・必読）

Genspark（Nano Banana 2）でフルページ漫画を作るときの**恒久ルール**。
目的は2つの再発防止：**①キャラの一貫性（特に髪色ブレ）**、**②セリフの誤植・文字崩れ**。
以降のすべての生成は、このガイドの「共通ブロック」を毎ページ先頭に付けて行う。

---

## 0. 崩れる理由と対策（要点）
- **髪色・服装がブレる** → `前と同じキャラで一貫` だけに頼ると1ページ単位でドリフトする。
  → 対策：**毎ページ、全登場キャラの配色を明示**し、`茶色にしない` のように**否定形で固定**する。
- **セリフが誤字・重複・意味不明になる** → セリフを情景説明文に混ぜるとAIが「描く文字」を区別できない。
  → 対策：**「描き込むテキスト」を独立ブロックに分離**、**短く・少なく（1ページ最大5フキダシ）・難漢字回避**。

---

## 2. 共通ブロック（毎ページの先頭に必ず貼る）
> 下記をそのままコピーし、末尾に「各コマ内容」と「描き込むテキスト」を続ける。

```
【作画ルール・厳守】縦長9:16の漫画1ページ、コマ枠で上から4つに区切る、フルカラーのアニメ漫画調、きれいな線画、透かし・英字ロゴなし。
【キャラ配色・絶対固定（毎回同一）】さくら＝鮮やかなオレンジ色のゆるふわボブ（茶色・赤茶にしない）＋大きな琥珀色の瞳＋濃紺スーツ＋白インナー／月島＝黒髪サイドパート＋銀縁眼鏡＋水色シャツ／鶴見部長＝白髪と白い口ひげ＋濃紺スリーピース＋恰幅／美咲＝栗色の低めポニーテール＋ベージュ系オフィスカジュアル／結衣＝明るい茶系セミロング＋濃紺スーツ／ケイタ＝電卓型マスコット＋まんまるの目＋ティール帽。※すべてのコマで髪色・目の色・服装を完全に一致させる。
【文字ルール・誤字厳禁】画面に描く日本語は、後述「描き込むテキスト」の文字だけを一字一句そのまま正確に描く。文字の増殖・重複・崩れ・意味不明な文字を禁止。難しい漢字は避け、読めない字は作らない。指定外の文字は一切描かない。フキダシは1ページ合計5個まで。
```

その後に：
```
【各コマの内容】コマ1…（情景・動き・表情。セリフはここに書かない）／コマ2…／コマ3…／コマ4…
【描き込むテキスト（この文字だけを正確に）】
コマ1 セリフ:「○○○」
コマ2 セリフ:「○○○」
コマ3 ナレーション:「○○○」
（無いコマは書かない）
```

**ポイント**
- 「各コマの内容」には**セリフを書かない**（情景だけ）。セリフは必ず「描き込むテキスト」に集約 → AIが描くべき文字を明確に認識でき、誤植が激減する。
- セリフは**1コマ1〜2個・各20字以内目安**。長い説明は地の文ナレーションを1つに絞る。
- 迷ったら**ひらがな多め**（例：「ぜんぶ」「ちゃんと」）。

## 3. 一貫性を上げる運用ステップ（推奨）
1. **各部（エピソード）の最初に「キャラクター設定シート」を1枚生成**（全キャラ勢揃い＋配色固定）。以降の生成で「前出の設定シートと同じ配色・髪型で」と一言添える。
2. Gensparkの**同じ会話**内で通しで生成する（会話をまたがない）。
3. 生成したら**1ページずつその場で確定回収**（`manga/` 取り込みフローと同じ）。過去画像の混入・順番ズレを防ぐ。
4. 髪色が茶色化した／誤字が出たページは、**そのページだけ再生成**（共通ブロックは変えず、`さくらの髪は必ず鮮やかなオレンジ` を一言強調で足す）。

---

## 4. さらに確実にしたい場合（将来オプション）
セリフ誤植を**ゼロ**にしたいなら、**フキダシを空（文字なし）で作画→サイト側のHTMLオーバーレイで文字を乗せる**方式が最強（`content/manga.js` のバブル機能）。
- 利点：誤字ゼロ・後から一括修正・多言語化が容易・キャラ作画にAIが集中できる。
- 欠点：バブル位置の調整が要る。
- 本ガイドの「文字ルール」で当面は焼き込み継続、量が増えたらこの方式へ移行を検討。
